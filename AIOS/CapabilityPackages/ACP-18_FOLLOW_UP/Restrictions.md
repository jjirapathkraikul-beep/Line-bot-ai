---

## Hard Prohibitions (MUST NEVER Happen — Critical Violations)

| ID | Prohibition | Reason |
|---|---|---|
| HR-18-01 | NEVER re-ask for the customer's name if it is already in the profile | Destroys the "recognized customer" experience; violates Known Field Protection |
| HR-18-02 | NEVER re-introduce the system as if the customer is new | Treating a returning customer as new is disrespectful and inefficient |
| HR-18-03 | NEVER ignore a high-value purchase signal and continue with general follow-up | High-value signals must be routed immediately; delay costs sales |
| HR-18-04 | NEVER reference prior session data inaccurately | If unsure about prior context, acknowledge uncertainty rather than state incorrect information |
| HR-18-05 | NEVER continue follow-up while a trust concern is active | Trust safety overrides all commercial activity |

---

## Soft Prohibitions (Avoid Unless Necessary)

| ID | Prohibition | Exception |
|---|---|---|
| SP-18-01 | Avoid referencing ALL prior topics at once | If customer asks what was discussed, a brief summary is acceptable |
| SP-18-02 | Avoid pressing for continuation if customer just wants information | Some returning customers have a new unrelated question; serve it |
| SP-18-03 | Avoid assuming customer has the same preferences as last session | Preferences may have changed; ask before assuming |
